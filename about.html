Intro

This app aims to be a user-friendly revision control client. It fits people working on their own projects, or that find git overwhelming.

The app uses "git" behind the scene, and is therefore fully compatible and can be used interchangeably together with other git softwares.

Start working - drop a folder

Dragging a project's folder onto the app's window makes it available for version control.

If the folder is not prepared (as a git repository), the user is asked to allow the program to initialize the folder for git.

Clickable items

Most things can be clicked. The icons means :
help
Show project folder
Settings dialog
Steps through the history of older descriptions
Steps towards newer descriptions

The title-bar texts are also clickable (see Advanced, below).

Store version with description

The status bar shows the number of changed files in the project. If all numbers are zero, you have no changed files. If one ore more of the numbers are non-zero (as in the figure), it shows that you are working.

When you feel you have reached an important point in your work (or are just afraid you will get lost if you continue) you may store a named revision of the project. The nice thing about the revision control system is, that you can later retrieve your whole project as it was at this point in time.

To store a revision of the changed files, a text description is filled in, and then the button is pressed.

Advanced

Repositories : You can have multiple active project folders (repositories). You switch by clicking the name (the first text in the title bar), to change to next project in the list.

Branches : You can have multiple branches of your work, that you can switch between. You switch by clicking the name (the second text in the title bar between parentheses), to change to next project in the list.
